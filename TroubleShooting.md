# 시행착오 및 문제 해결 보고서

## 1. 개발 환경 설정 오류

- 문제  
FastAPI 실행 시 가상환경(.venv) 경로 혼동으로 서버 실행 실패

- 원인  
가상환경은 반드시 직접 활성화해야 한다는 잘못된 인식

- 해결  
`uv run fastapi run` 명령어를 통해 환경 자동 연결

- 배운 점  
개발 환경 실행 방식에 대한 이해 부족이 문제의 핵심이었으며,  
도구 기반 실행 방식에 대한 이해가 필요함을 확인

---

## 2. Spark 처리 방식 선택 오류

- 문제  
SQL View 기반 처리 방식만 고려하여 비효율 발생

- 원인  
Spark DataFrame API에 대한 이해 부족

- 해결  
DataFrame API 기반으로 전환 (groupBy, agg 등 활용)

- 배운 점  
Spark는 SQL 방식과 DataFrame 방식 모두 가능하며,  
상황에 따라 더 직관적이고 효율적인 방법 선택 필요

---

## 3. 데이터 품질 이슈 (날짜 포맷 불일치)

- 문제  
날짜 형식 혼재 (YYYY-MM-DD vs DD/MM/YYYY)

- 원인  
CSV 변환 과정에서 포맷 불일치 발생

- 해결  
데이터 전처리 단계에서 포맷 통일

- 배운 점  
문제 발생 시 코드보다 원본 데이터 검증이 우선

---

## 4. 데이터 처리 구조 설계 변경

- 문제  
Spark 실시간 처리 구조 → 성능 및 시간 문제 발생

- 원인  
데이터 규모 대비 과도한 설계

- 해결  
사전 집계 후 DB 적재 구조로 변경

- 배운 점  
이론보다 성능과 응답속도 중심 설계 필요

---

## 5. 컬럼 설계 오류

- 문제  
하나의 컬럼에 복수 의미 혼합

- 원인  
데이터 모델링 단계 검증 부족

- 해결  
컬럼 분리 (단일 책임 원칙 적용)

- 배운 점  
스키마 설계는 초기 단계에서 반드시 검증 필요

---

## 6. NULL vs 0 처리 기준 부재

- 문제  
집계 결과 왜곡 발생

- 원인  
데이터 정의 기준 미정립

- 해결  
NULL / 0 기준 명확화

- 배운 점  
데이터 정의 기준은 사전에 반드시 확립 필요

---

## 7. 데이터 타입 및 쿼리 최적화

- 문제  
날짜를 TEXT로 저장 → 성능 저하

- 해결  
DATE 타입 변경 및 JOIN 구조 개선

- 배운 점  
데이터 타입과 쿼리 구조는 성능에 직접 영향

---

## 8. 환경 불일치 문제

- 문제  
Python 버전 차이로 실행 오류

- 해결

- 배운 점  
환경 표준화 중요성 확인

## 9. 대용량 데이터 처리 방식 개선

- 문제

- 해결

- 배운 점

---

## 10. 데이터 파이프라인 오류 전파

- 문제

- 배운 점
