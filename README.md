# 🚇 서울교통공사 지하철 데이터 분석 프로젝트

## 1. 프로젝트 개요

본 프로젝트는 서울교통공사 지하철 승하차 데이터를 기반으로  
단순 집계를 넘어 **유동 인구 패턴 및 공간적 특성 분석**을 수행하는 것을 목표로 한다.

특히, Spark 기반 데이터 처리와 API 서버를 통해  
데이터 분석 결과를 효율적으로 제공하는 시스템을 구축하였다.

---

## 2. 프로젝트 목표

- 대용량 지하철 데이터 정제 및 집계
- 유의미한 핵심 지표 설계 (순유입, 혼잡도 등)
- API 기반 데이터 제공 구조 구축
- 시각화를 통한 인사이트 도출

---

## 3. 데이터 설명

- 출처: 서울교통공사
- 구성:
  - 역별 승차/하차 인원
  - 날짜 및 시간 정보

### ⚠ 데이터 특징
- Wide Format 구조
- 날짜 포맷 불일치 존재
- NULL / 0 값 혼재

---

## 4. 시스템 아키텍처
CSV → Spark → DB → FastAPI 


### 사용 기술
- Python
- Apache Spark
- FastAPI
- MariaDB

---

## 5. 데이터 처리 흐름
원본 데이터 → 정제 → 집계 → DB 저장 → API → 시각화


---

## 6. 핵심 지표

### 1) 순유입 (Net Flow)
- 정의: 승차 - 하차
- 의미:
  - + : 유입 지역
  - - : 유출 지역

### 2) 시간대 집중도
- 특정 시간대 유동인구 집중 정도

---

## 7. 주요 기능

- 역별 순유입 조회 API
- 시간대별 유동인구 분석
- 시각화 대시보드 제공

---

## 8. 결과 및 인사이트

- 특정 지역은 출근 시간 유입 ↑ / 퇴근 시간 유출 ↑
- 상업지역과 주거지역 패턴 차이 확인

---

## 9. 한계 및 개선 방향

- 실시간 데이터 미반영
- 데이터 품질 이슈 존재
- 향후 머신러닝 기반 예측 가능

---
